# HOME SCREEN WIDGET

---

## WIDGET TYPES — TWO SIZES

### Small Widget (2×2)
```
┌─────────────────────┐
│ THE IMPERIUM        │
│                     │
│ G: OFF DUTY         │
│ H: NIGHT SHIFT      │
│                     │
│ Next: Lunch 1PM     │
│ 💧 64/100 oz  [+]   │
└─────────────────────┘
```

### Medium Widget (4×2)
```
┌───────────────────────────────────────┐
│ THE IMPERIUM          Wed Jan 14      │
│ G: OFF DUTY    H: NIGHT — 7P–7A      │
│                                       │
│ 1:00 PM  Lunch — Turkey Avocado Wrap  │
│ 6:00 PM  Evening review opens  🔔     │
│                                       │
│ 🍽  Tonight: Sheet Pan Chicken        │
│    ⚠ Thaw chicken thighs now         │  ← prep flag if recipe has thaw note
│                                       │
│ 💧 64 / 100 oz  ████░    [+ Log]     │
└───────────────────────────────────────┘
```

**Tonight's Dinner pinned slot:**
- The medium widget always shows tonight's dinner in a dedicated row.
- If the recipe card has a prep note that includes "thaw", "marinate", "slow cooker",
  or "start by [time]", the widget shows a ⚠ prep flag with the specific action.
- The prep flag text is pulled directly from the recipe's `batchNote` or `prepNote`
  field — not generated by AI, not inferred. Only shows if the field contains one
  of the trigger words above.
- Tapping the dinner row opens the app to that recipe's inline expansion on the planner.
- If it's past 7PM, the dinner row shows "Dinner complete?" with a checkmark option.

## WIDGET CONTENT

| Element | Imperium | Tending |
|---------|---------|---------|
| Header | "The Imperium" | "The Tending" |
| Shift badge | G: status / H: status | H: status / G: status |
| Next 2-3 planner items | Shows times + labels | Shows times + labels |
| Hydration | Current oz / target + tap-to-log | Current oz / target + tap-to-log |
| Alarm indicators | 🔔 on items with alarms | 🔔 on items with alarms |

---

## WIDGET INTERACTIONS

- Tap anywhere on widget → opens app at Planner tab, Today view
- Tap "+ Log" hydration → opens app at hydration tracker
- Tap a specific planner item → opens app at that item

## TECHNICAL IMPLEMENTATION

```typescript
// Android widget using react-native-widget-extension
// Widget refreshes every 15 minutes (Android minimum interval)
// Widget data served from a WorkManager background task

// widget/SovereignWidget.tsx
export default function SovereignWidget() {
  const profile = useWidgetProfile();    // reads from AsyncStorage
  const planner = useWidgetPlanner();   // next 3 items
  const hydration = useWidgetHydration(); // today's oz
  const shifts = useWidgetShifts();     // today's shift status for both

  return (
    <FlexWidget>
      <TextWidget text={profile.displayName} style={styles.header} />
      <ShiftBadgeWidget garrinStatus={shifts.imperium} holliStatus={shifts.tending} />
      <PlannerItemsList items={planner.nextItems} />
      <HydrationRow current={hydration.logged} target={hydration.target} />
    </FlexWidget>
  );
}
```

## WIDGET UPDATE TRIGGER

Widget updates when:
- App opens (forces refresh)
- Hydration is logged (immediate)
- Alarm fires (forces refresh)
- Every 15 minutes (background task)
- Shift status changes (Firebase listener triggers)
